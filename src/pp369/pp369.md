## 1. Nguyên tắc chiến lược

PP369 dựa trên quan sát: giá crypto thường **phản ứng tại các mốc là bội số của 3** so với giá đầu tháng.

### Xây dựng lưới mốc

```
Bước 1: Lấy nến H1 đầu tiên của tháng (00:00 UTC ngày 1)
         → Open và Close của nến đó là 2 mốc gốc

Bước 2: Tạo lưới mốc bằng cách cộng/trừ bội số của bước
         Bước = 3 × đơn vị giá (tự động theo mức giá)

Bước 3: Chia làm 2 dãy:
         dãy "tren" = từ max(Open, Close) → mốc LONG entry
         dãy "duoi" = từ min(Open, Close) → mốc SHORT entry
```

### Bước giá theo từng mức giá

Bước được tính từ **giá hiện tại** (không phải H1 open) — đảm bảo khoảng cách mốc luôn phù hợp ngay cả khi coin đã tăng/giảm mạnh so với đầu tháng.

| Mức giá hiện tại | Đơn vị | Bước (×3) | Step/Price | Ví dụ                         |
|------------------|--------|-----------|------------|-------------------------------|
| ≥ $10,000        | 1,000  | 3,000     | ~3%        | BTC: mốc ±3000, ±6000, ±9000 |
| $1,000–$9,999    | 100    | 300       | 3–30%      | ETH: mốc ±300, ±600, ±900    |
| $100–$999        | 10     | 30        | 3–30%      | BNB: mốc ±30, ±60, ±90       |
| $10–$99          | 1      | 3         | 3–30%      | token $30: mốc ±3, ±6, ±9    |
| $1–$9.99         | 0.1    | 0.3       | 3–30%      | UNI: mốc ±0.3, ±0.6, ±0.9   |
| $0.20–$0.99      | 0.01   | 0.03      | 3–15%      | ADA: mốc ±0.03, ±0.06        |
| $0.02–$0.19      | 0.001  | 0.003     | 1.5–15%    | HBAR: mốc ±0.003, ±0.006     |
| $0.002–$0.019    | 0.0001 | 0.0003    | ~1.5–15%   |                               |
| < $0.002         | 0.00001| 0.00003   | ~1.5%      |                               |

> **Lý do dùng giá hiện tại:** Nếu dùng H1 open price, coin tăng 10x trong tháng sẽ có step nhỏ hơn 10x so với cần thiết → mốc dày đặc → signal liên tục mỗi vài phút (bug VELVET tháng 6/2026).

## 2. Logic phát tín hiệu

### Điều kiện tín hiệu LONG

```
✅ Giá đang tiếp cận mốc "tren" bên DƯỚI giá (trong vòng 0.1%)
✅ Lần chạm mốc gần nhất là mốc "duoi" bên TRÊN (lastSide = 'upper')
   → Nghĩa là: giá đã đi từ mốc trên xuống, đang về mốc dưới → vào LONG
```

### Điều kiện tín hiệu SHORT

```
✅ Giá đang tiếp cận mốc "duoi" bên TRÊN giá (trong vòng 0.1%)
✅ Lần chạm mốc gần nhất là mốc "tren" bên DƯỚI (lastSide = 'lower')
   → Nghĩa là: giá đã đi từ mốc dưới lên, đang về mốc trên → vào SHORT
```

### Quy tắc 3 lần (độ mạnh tín hiệu)

| Lần thứ | Độ mạnh | Điểm Confluence |
|---------|---------|-----------------|
| 1       | strong  | +2 điểm         |
| 2       | medium  | +1 điểm         |
| 3+      | weak    | +1 điểm (mốc dễ phá) |

> **Ghi chú:** Lần 3 trở đi vẫn alert nhưng mốc đã bị chạm nhiều → xác suất phá mốc cao hơn.

### Đếm số lần roundtrip (analyzeRoundtrips)

Thuật toán quét toàn bộ nến 1m từ đầu tháng:
- `lowerCount` = số lần giá đi từ shortEntry (mốc trên) xuống longEntry (mốc dưới) → đây là số lần LONG đã xảy ra
- `upperCount` = số lần giá đi từ longEntry lên shortEntry → số lần SHORT đã xảy ra
- Nến chạm cả 2 mốc trong 1 cây: dùng hướng nến (bearish/bullish) để xác định bên nào là điểm cuối

> **Quan trọng:** Trước khi phân tích, nến đang hình thành (nến cuối cùng) bị loại ra bằng `done = m1Candles.slice(0, -1)`. Lý do: nến chưa đóng có thể chạm mốc tạm thời rồi đảo lại → nếu tính vào sẽ gây tín hiệu giả. Chỉ nến đã đóng hoàn chỉnh mới được dùng để xác định `lastSide` và đếm roundtrip.

## 3. Hệ thống cache

PP369 dùng 3 lớp cache in-memory để giảm API call:

### H1 Cache — nến đầu tháng
```
Key: symbol
Value: { monthStart, openPrice, closePrice, step, decimals, upperPrice, lowerPrice }
TTL: tự động reset khi sang tháng mới (so sánh monthStart)
API call: 1 lần/tháng/coin
```

### 1m Cache — nến incremental
```
Key: symbol
Value: { monthStart, candles: [...], cursor: openTime }
TTL: tự động reset khi sang tháng mới
API call: lần đầu fetch toàn bộ từ đầu tháng (~300-1500 nến), sau đó chỉ fetch nến mới
Overlap: nến cuối luôn được re-fetch để cập nhật nến đang hình thành
```

### Level Cache — cho WebSocket filter
```
Key: symbol
Value: { longEntry, shortEntry, step }
Cập nhật: sau mỗi lần get369Signal() thành công
Dùng bởi: getNearbySymbols() để lọc coin trước khi scan
  step: effectiveStep tại thời điểm tính — dùng để tính adaptive threshold
```

### Hiệu năng
| Lần đầu (cold start) | Lần sau (warm cache) |
|----------------------|----------------------|
| ~47 giây (20 coin)   | ~1 giây (chỉ fetch nến mới) |

## 4. WebSocket proximity filter

Thay vì scan toàn bộ coin list mỗi phút, PP369 dùng WebSocket mark price (~3s/update) để chỉ scan coin đang **gần mốc**.

```
Bot khởi động
  → start369Stream(symbols)       subscribe markPrice cho tất cả coin
  → Mỗi 3 giây: cập nhật _prices[symbol]

Mỗi 1 phút (run369AlertScan):
  → getNearbySymbols()            lọc coin có giá trong vùng threshold của mốc
  → get369SignalsForCoins(nearby) chỉ fetch nến cho coin gần mốc
  → Coin xa mốc: bỏ qua hoàn toàn
```

### Đo khoảng cách từ giá hiện tại đến mốc

```
distLong  = (price - longEntry)  / price   ← giá cách mốc LONG bên dưới bao nhiêu %
distShort = (shortEntry - price) / price   ← giá cách mốc SHORT bên trên bao nhiêu %

Coin được scan khi: distLong ≤ threshold  HOẶC  distShort ≤ threshold
```

Ví dụ: giá hiện tại 0.79927, longEntry = 0.78830, shortEntry = 0.84779
```
distLong  = (0.79927 - 0.78830) / 0.79927 ≈ 1.37%  ✅ < threshold → scan
distShort = (0.84779 - 0.79927) / 0.79927 ≈ 6.07%  ❌ > threshold → skip
```

### Adaptive threshold

Threshold **không cố định 2%** mà tự điều chỉnh theo step/price ratio của từng coin:

```javascript
adaptiveThreshold = Math.max(0.02, (levels.step / price) * 0.6)
```

Lý do: với coin có step lớn (SOL step=20% của giá), nếu dùng 2% cố định thì coin bị bỏ qua khi đang ở giữa 2 mốc (midpoint cách mốc ~10%, >> 2%).

| Coin | Step/Price | Threshold cũ | Threshold mới |
|------|-----------|--------------|---------------|
| BTC  | 3%        | 2%           | 2% (không đổi) |
| ETH  | 10%       | 2%           | 6% → luôn scan |
| SOL  | 20%       | 2%           | 12% → luôn scan |
| DOGE | 10%       | 2%           | 6% → luôn scan |
| ADA  | 3.75%     | 2%           | 2.25% → luôn scan |

**Fail-open:** Coin chưa có giá WebSocket hoặc chưa có level cache → luôn được scan (không bỏ sót tín hiệu).

## 6. API reference

### Core strategy

#### `get369Signal(symbol, currentPrice?)`
Lấy tín hiệu PP369 cho 1 coin.

```javascript
const sig = await get369Signal('BTC');
const sig = await get369Signal('ETH', 3500.5); // truyền giá sẵn, bỏ qua giá nến cuối

// Kết quả trả về:
{
  symbol:       'BTC',
  signal:       'LONG' | 'SHORT' | 'NONE',
  strength:     'strong' | 'medium' | 'weak' | 'none',
  touchCount:   0,           // số lần loại này đã xảy ra trước (0 = lần đầu)
  targetLevel:  58855,       // mốc entry
  condLevel:    61653.2,     // mốc điều kiện (đã chạm trước đó)
  nearestAbove: 61653.2,     // mốc ngay trên giá
  nearestBelow: 58855,       // mốc ngay dưới giá
  currentPrice: 60500,
  openPrice:    96012.5,     // Open H1 đầu tháng
  closePrice:   95987.3,     // Close H1 đầu tháng
  step:         3000,        // bước giá
  month:        '2026-06',
  reason:       '[369] ...',
  debugInfo: {
    lowerCount:   1,         // số lần LONG đã xảy ra
    upperCount:   0,         // số lần SHORT đã xảy ra
    lastSide:     'upper',   // 'lower' | 'upper' | null
    pairLow:      58855,
    pairHigh:     61653.2,
    totalCandles: 8640,
  },
  nearLevels: [              // các mốc trong vòng ±5.5 bước quanh giá
    { value: 58855, type: 'tren', tier: -1 },
    ...
  ]
}
```

#### `get369SignalsForCoins(symbols, taMap?)`
Batch scan nhiều coin song song.

```javascript
const signals = await get369SignalsForCoins(
  ['BTC', 'ETH', 'SOL'],
  { BTC: { price: 60500 }, ETH: { price: 3400 } } // taMap: truyền giá sẵn (optional)
);
// Trả về: { BTC: signal, ETH: signal, SOL: signal }
```

#### `score369Method(sig369, direction)`
Tính điểm đóng góp cho Confluence Scorer.

```javascript
const { score, reasons } = score369Method(sig, 'LONG');
// score: 0 | 1 | 2
// reasons: ['[PP369] LONG lần 1 tại 58855 ← từ 61653.2 (strong) (+2)']
```

#### `format369ForPrompt(signals369Map)`
Tạo text block nhúng vào Gemini prompt.

```javascript
const text = format369ForPrompt(signals);
// Trả về string mô tả tất cả tín hiệu cho AI đọc
```

#### `getLevelCache()`
Lấy level cache hiện tại (dùng cho WebSocket filter).

```javascript
const cache = getLevelCache();
// { BTC: { longEntry: 58855, shortEntry: 61653.2, step: 3000 }, ... }
```

#### `PROXIMITY_PCT`
Ngưỡng proximity filter tối thiểu: `0.02` (2%). Threshold thực tế sẽ lớn hơn nếu step/price > 3.3% — xem [Adaptive threshold](#adaptive-threshold).

### WebSocket stream

#### `start369Stream(symbols)`
Khởi động / cập nhật stream. Gọi lại với list mới → tự reconnect.

```javascript
start369Stream(['BTC', 'ETH', 'SOL', 'BNB']);
```

#### `stop369Stream()`
Dừng stream, giải phóng kết nối.

#### `getMarkPrice(symbol)`
Giá markPrice hiện tại từ stream (null nếu chưa có data).

```javascript
const price = getMarkPrice('BTC'); // 60500.5 hoặc null
```

#### `getNearbySymbols(symbols, levelCache, threshold?)`
Lọc coin đang gần mốc. `threshold` là ngưỡng tối thiểu (mặc định 0.02 = 2%) — threshold thực tế sẽ tự điều chỉnh lớn hơn nếu step/price > 3.3%.

```javascript
const nearby = getNearbySymbols(
  ['BTC', 'ETH', 'SOL'],
  getLevelCache(),
  0.02   // threshold tối thiểu — adaptive sẽ tự tăng cho coin có step lớn
);
// Trả về subset coin đang trong vùng gần mốc (tính từ giá hiện tại đến mốc)
```

### Signal log

#### `logSignal369(sig)`
Ghi tín hiệu LONG/SHORT vào `data/369_signals.jsonl`.

```javascript
logSignal369(sig); // chỉ ghi khi sig.signal === 'LONG' | 'SHORT'
```

File log mỗi dòng là 1 JSON:
```json
{"ts":"2026-06-09T14:32:10","symbol":"BTC","month":"2026-06","signal":"LONG","price":58900,"h1Open":96012.5,"h1Close":95987.3,"targetLevel":58855,"condLevel":61653.2,"touchCount":0,"lan":1,"strength":"strong","pairLow":58855,"pairHigh":61653.2}
```

Fields đầy đủ:
| Field        | Nguồn                     | Mô tả                              |
|--------------|---------------------------|------------------------------------|
| `ts`         | `new Date()` (VN timezone)| Timestamp tín hiệu                 |
| `symbol`     | `sig.symbol`              | Ký hiệu coin                       |
| `month`      | `sig.month`               | Tháng phân tích (YYYY-MM)          |
| `signal`     | `sig.signal`              | `LONG` hoặc `SHORT`                |
| `price`      | `sig.currentPrice`        | Giá lúc phát tín hiệu              |
| `h1Open`     | `sig.openPrice`           | Open nến H1 đầu tháng — mốc gốc   |
| `h1Close`    | `sig.closePrice`          | Close nến H1 đầu tháng — mốc gốc  |
| `targetLevel`| `sig.targetLevel`         | Mốc entry                          |
| `condLevel`  | `sig.condLevel`           | Mốc điều kiện (đã chạm trước đó)   |
| `touchCount` | `sig.touchCount`          | Số lần đã xảy ra trước (0 = lần 1) |
| `lan`        | `touchCount + 1`          | Lần thứ mấy (1-indexed)            |
| `strength`   | `sig.strength`            | `strong` / `medium` / `weak`       |
| `pairLow`    | `sig.debugInfo.pairLow`   | Mốc dưới của cặp đang scan         |
| `pairHigh`   | `sig.debugInfo.pairHigh`  | Mốc trên của cặp đang scan         |

#### `loadSignalHistory()`
Đọc toàn bộ lịch sử tín hiệu.

```javascript
const history = loadSignalHistory();
// Trả về mảng object
```

// Dừng stream khi thoát

### Ví dụ: scan định kỳ mỗi phút

```javascript
const pp369 = require('./src/pp369');

const SYMBOLS = ['BTC', 'ETH', 'SOL', 'BNB', 'XRP', 'DOGE'];

pp369.start369Stream(SYMBOLS);

setInterval(async () => {
  // Chỉ scan coin đang gần mốc
  const nearby = pp369.getNearbySymbols(SYMBOLS, pp369.getLevelCache(), pp369.PROXIMITY_PCT);
  if (!nearby.length) return;

  const signals = await pp369.get369SignalsForCoins(nearby);

  for (const [sym, sig] of Object.entries(signals)) {
    if (sig.signal === 'LONG' || sig.signal === 'SHORT') {
      pp369.logSignal369(sig);
      // Gửi Telegram, webhook, v.v...
      console.log(pp369.format369Alert([sig]));
    }
  }
}, 60_000);
```

## 8. Chạy standalone

PP369 có thể chạy như 1 script độc lập, không cần bot Telegram hay AI.

### Test tín hiệu từ terminal (news-noti có sẵn script)

```bash
# Test 8 coin mặc định
node scripts/test-369.js

# Test coin tuỳ chọn
node scripts/test-369.js BTC ETH UNI ADA
```

Output terminal có màu, hiển thị đầy đủ:
- Nến H1 đầu tháng (Open/Close/Bước)
- Vị trí giá so với lưới mốc
- Số lần roundtrip (lowerCount/upperCount)
- Điều kiện LONG/SHORT đạt hay không
- Bảng mốc lân cận ±5 bước
- Điểm cộng vào Confluence Scorer

### Script tối giản chạy độc lập

```javascript
// scan.js — không cần Telegram, không cần AI
require('dotenv').config(); // chỉ cần nếu dùng TIMEZONE từ .env

const pp369 = require('./src/pp369');

async function main() {
  const symbols = ['BTC', 'ETH', 'SOL'];
  console.log('Đang lấy tín hiệu 369...');

  const signals = await pp369.get369SignalsForCoins(symbols);

  for (const [sym, sig] of Object.entries(signals)) {
    const icon = sig.signal === 'LONG' ? '🟢' : sig.signal === 'SHORT' ? '🔴' : '⚪';
    console.log(`${icon} ${sym}: ${sig.signal} | ${sig.reason}`);
    if (sig.signal !== 'NONE') {
      pp369.logSignal369(sig);
    }
  }

  process.exit(0);
}

main().catch(console.error);
```

## 9. Cấu hình

| Biến môi trường | Mặc định           | Mô tả                           |
|-----------------|--------------------|---------------------------------|
| `TIMEZONE`      | `Asia/Ho_Chi_Minh` | Timezone cho signal log         |

Các hằng số trong `core.js` (thay đổi bằng cách sửa file):

| Hằng số          | Mặc định | Mô tả                                                         |
|------------------|----------|---------------------------------------------------------------|
| `LEVELS_RANGE`   | 6        | Số tầng mốc tạo ra mỗi hướng (±6), tự mở rộng nếu giá xa H1 |
| `TOUCH_TOLERANCE`| 0.003    | 0.3% dung sai để tính "đã chạm mốc" trong analyzeRoundtrips  |
| `NEAR_LEVEL_PCT` | 0.001    | 0.1% vùng tiếp cận mốc để phát alert                        |
| `PROXIMITY_PCT`  | 0.02     | 2% ngưỡng tối thiểu WebSocket filter — adaptive sẽ tự tăng  |

## 10. Điểm cộng vào Confluence Scorer

`score369Method(sig, direction)` trả về điểm tích hợp vào hệ thống chấm điểm:

```
signal LONG/SHORT khớp direction → +2 (strong) | +1 (medium) | +1 (weak)
signal NONE hoặc ngược chiều    →  0
```

Thang điểm tổng Confluence (tham khảo):

| Tổng điểm | Verdict          | Ý nghĩa                    |
|-----------|------------------|----------------------------|
| < 5       | NO_SETUP         | Không đủ điều kiện         |
| 5–6       | CAUTIOUS         | Thận trọng, size nhỏ       |
| 7–8       | CONFIDENT        | Tốt, vào lệnh bình thường  |
| 9–10      | BEST_OPPORTUNITY | Tín hiệu mạnh nhất         |
| > 10      | BEST_OPPORTUNITY | Cực kỳ hiếm                |

PP369 đóng góp tối đa 2 điểm trong thang điểm này.
